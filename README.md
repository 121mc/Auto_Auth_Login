# NJU Auto Auth Login

面向南京大学统一身份认证与选课系统的浏览器自动登录扩展。

扩展可自动填写学号/工号和密码，并在浏览器本地使用 ONNX 模型识别验证码。它支持访问统一身份认证页面时自动登录、定时检查认证状态，以及进入选课系统时自动处理点选验证码。

> [!IMPORTANT]
> 本项目为非官方工具，仅供学习与个人使用。请自行评估账号安全、学校相关规定及页面变更带来的风险。

## 功能

- 统一身份认证页自动登录
- 每隔 3–5 分钟随机检查一次登录状态
- 登录失效后自动打开认证页并重新登录
- 选课系统登录页自动填写账号、识别并点击验证码
- 普通字符验证码与中文点选验证码均在本地识别
- Popup 面板展示当前状态、下次检查时间和运行日志
- 所有模型和推理依赖均随扩展提供，无需调用第三方验证码服务

## 支持范围

| 场景 | 地址 | 功能 |
| --- | --- | --- |
| 南京大学统一身份认证 | `https://authserver.nju.edu.cn/authserver/login*` | 自动填写、识别字符验证码并提交 |
| 南京大学选课系统 | `https://xk.nju.edu.cn/xsxkapp/sys/xsxkapp/*default/index.do` | 自动填写、识别中文点选验证码并登录 |

扩展使用 Manifest V3，适用于支持加载解压扩展的 Chromium 浏览器，例如 Chrome 和 Edge。

## 安装

本项目无需编译，也无需安装 npm 或 Python 依赖。

1. 下载本项目源码并解压，确保目录中直接包含 `manifest.json`。
2. 打开浏览器扩展管理页：
   - Chrome：`chrome://extensions/`
   - Edge：`edge://extensions/`
3. 开启「开发者模式」。
4. 点击「加载已解压的扩展程序」。
5. 选择本项目根目录。
6. 建议将扩展固定到浏览器工具栏，便于配置和查看状态。

更新源码后，在扩展管理页点击该扩展的「重新加载」即可应用更改。

## 使用方法

1. 点击工具栏中的扩展图标。
2. 输入南京大学统一身份认证的学号/工号和密码。
3. 点击「保存学号和密码」。
4. 根据需要开启功能：
   - **统一身份认证自动登录**：访问认证页时自动完成登录。
   - **定时检查登录情况**：周期性检查认证状态；该选项依赖统一身份认证自动登录。
   - **选课页面自动登录**：进入支持的选课登录页时自动登录。
5. 可在面板底部查看运行日志和下一次检查时间。

开启定时检查后，扩展会立即执行一次状态检查，之后以 3–5 分钟的随机间隔继续运行。浏览器必须保持运行，后台任务才能正常触发。

## 工作原理

### 统一身份认证

1. Background Service Worker 使用 `chrome.alarms` 安排状态检查。
2. 扩展请求认证登录页，并根据响应判断当前会话是否仍然有效。
3. 会话失效时，扩展在后台打开统一身份认证登录页。
4. Content Script 填写账号和密码，读取验证码图片。
5. Offscreen Document 使用 ONNX Runtime Web 和本地 OCR 模型识别验证码。
6. 扩展回填验证码并提交表单；验证码错误时会刷新并重试。
7. 登录成功后更新状态，并关闭由扩展打开的登录标签页。

### 选课系统

1. Content Script 检测选课登录页面并填写凭据。
2. 扩展截取中文点选验证码。
3. 检测模型定位候选文字，OCR 模型识别提示文字和候选字。
4. 扩展计算目标字符坐标，按要求的顺序模拟点击。
5. 提交登录；若验证码错误，则刷新后重试。

## 安全与隐私

- 学号/工号和密码保存在 `chrome.storage.local` 中。
- 凭据和验证码不会发送给第三方识别服务。
- 验证码图片、检测和 OCR 推理均在浏览器本地完成。
- 扩展只声明统一身份认证与选课系统所需的站点访问权限。

需要注意，`chrome.storage.local` 并不是专用密码保险箱。任何能够读取该浏览器配置文件或扩展本地数据的人，都可能接触到保存的凭据。请只在可信的个人设备和浏览器配置中使用，不建议在公用电脑上保存密码。

卸载扩展会清除其本地存储。若只想暂停功能，可以关闭三个自动登录开关。

## 常见问题

### 状态一直显示“未配置”

确认已经填写学号/工号和密码，并点击「保存学号和密码」。

### 没有执行定时检查

「定时检查登录情况」只有在「统一身份认证自动登录」开启时才可用。还需要保持浏览器运行，并确认扩展没有被禁用。

### 自动登录或验证码识别失败

- 打开扩展面板查看运行日志。
- 手动刷新目标登录页后重试。
- 在扩展管理页重新加载扩展。
- 确认学校登录页面地址仍在“支持范围”中。

学校页面结构或验证码样式发生变化时，扩展可能需要同步更新。

### 选课系统没有自动登录

确认「选课页面自动登录」已开启，并检查当前网址是否匹配支持的选课系统地址。

## 开发与调试

项目使用原生 HTML、CSS 和 JavaScript，无构建步骤。

- Popup 日志：右键扩展面板，选择「检查」。
- Service Worker 日志：在扩展管理页打开该扩展的 Service Worker 检查窗口。
- Content Script 日志：在统一身份认证页或选课页打开开发者工具。
- 修改代码后：回到扩展管理页点击「重新加载」，再刷新目标页面。

开发时请勿提交真实账号、密码、Cookie 或包含个人信息的调试日志。
